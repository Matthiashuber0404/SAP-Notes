1. Einstieg

Das DDIC ist das zentrale Werkzeug zur Daten-Definition, insbes. auch zur Definition von Datenbanktabellen. Das DDIC ist vollständig in die ABAP-Workbench integriert, d.h.Zugriff aus jeder einzelnen Komponente, bzw. von jedem ABAP-Programm aus ist möglich.

Das DDIC verwaltet nur Definitionen und hält keine Datensätze. Diese befinden sich auf der echten Datenbank.

Der Aufruf des DDIC erfolgt über die Transaktion SE11.

Figure 1. ABAP Dictonary: Einstieg (SE11)

Dies ist eine sehr mächtige Transaktion. Hier können neben Datenbanktabellen weitere Elemente angelegt oder bearbeitet werden.

2. Erstellen von Datenbanktabellen

2.1. Übersicht

Eine Tabelle besteht aus verschiedenen Feldern. Ein Feld ist zusammengesetzt aus einem Datenelement (enthält Feldbezeichnungen usw.) und braucht eine Domäne. Diese beschreibt die technischen Charakteristika, z.B. Datentypen und –längen.

Domänen und Datenelemente sind wiederverwendbar. Eine Domäne kann in mehr als einem Datenelement verwendet werden und ein Datenelement in mehr als einem Feld und einer Tabelle.

Vorteil: Wenn die Länge eines Feldes geändert werden muss, braucht man diese nur in der Domäne ändern. Die Änderung wird in allen Feldern wirksam, die diese Domäne benutzen.

In jedem SAP-System existieren mehr als 13.000 Domänen, die von SAP angelegt wurden.

Namenskonventionen: für DDIC-Elemente, die Sie anlegen, bitte immer ZXXX am Anfang, Z für benutzerdefiniertes Feld und XXX für Ihr Kürzel.

2.3. Domäne

Die Grundlage für die Definition von Tabellenfeldern sind Domänen. Sie enthalten die technischen Eigenschaften wie Datentyp und Länge. Zusätzlich kann der Wertebereich noch weiter eingeschränkt werden.

Der Definition liegen Dictionary-Typen zugrunde (zu unterscheiden von den Typen, die im Programm verwendet werden).

Figure 2. Beispiel-Domäne

Einer Domäne kann ein Wertebereich zugeordnet werden.

Figure 3. Wertebereich: Intervall von 1-999
Tip
 Nicht für jede Komponente der DB-Tabelle muss eine Domäne angelegt werden. Eine solche ist nur dann sinnvoll, wenn man beispielsweise einen Wertebereich festlegen will. Das System beeinhaltet mehrere Tausend an fertigen Domänen, die einfach verwendet werden können, z.B. CHAR5, CHAR20, INT1, INT2, DEC10_2, DATS, usw.

In unserem Fall werden für die anderen Komponeten keine weiteren Domänen angelegt.

2.4. Datenelement

Ein Datenelement greift auf die technischen Eigenschaften einer Domäne zurück und ergänzt diese mit betriebswirtschaftlichen oder programmspezifischen Informationen (sog. semantische Eigenschaften).

Datenelemente und Domänen besitzen getrennte Namensräume, sie können daher identische Namen besitzen.

Die Anlage erfolgt über das Grundbild SE11 über die Obergruppe Datentyp. Anschließend wird ein Name (in Ihrem Namensbereich) eingegeben.

Zunächst wird ein Datenelement für die Personen-ID angelegt. Die vorher angelegte Domäne wird verwendet.

2.5. Transparente Tabelle

Nach Bereitstellung aller erforderlichen Datenelemente und Domänen kann eine Tabelle erzeugt werden. Eine Tabelle ist ein reales Objekt auf der Datenbank.

Eine transparente Tabelle hat eine Eins-zu-Eins-Beziehung zu einer Tabelle in der Datenbank. Ihre Struktur entspricht einer einzelnen Datenbanktabelle. Sie werden für Anwendungsdaten (Stamm- oder Bewegungsdaten) benutzt.

Die Anlage erfolgt analog von Domänen oder Datenelementen.

Es soll nun die Datenbanktabelle ZXXX_XX_T_PERSON angelegt werden.

3. Data Browser

Um Daten anzeigen zu lassen bzw. zu bearbeiten, gibt es zwei Möglichkeiten:

Diese Transaktion ist auch über die Datenbanktabelle über das Menü (Mehr-Hilfsmittel-Tabelleninhalt) erreichbar, bzw. über Strg-Shift-F10.

6. Währungs- und Mengenfelder

Es gibt zwei spezielle Datentypen, die zusätzlich ein weiteres Feld benötigen, um dieses zu spezifizieren.

1) Währung
Geldbeträge werden immer in ihrer Währung angegeben, z.B. 100 Euro, 200 USD, usw.

Das bedeutet, dass in der Datenbanktabelle immer ein Feld für die Währung vorhanden sein muss.

Note
 In der SAP-Tabelle TCURC sind alle gültigen Währungscodes enthalten.

2) Mengeneinheiten
Zu jeder Mengenangabe ist auch eine Einheit erforderlich, z.B. 10 Meter, 5 Liter, 2 Tonnen, usw.

Note
 In der SAP-Tabelle T006 befinden sich alle gültigen Maßeinheiten.

Erweitern Sie die Tabelle PERSON um die Felder Gehalt und Größe, jeweils mit den dazugehörigen Feldern.
