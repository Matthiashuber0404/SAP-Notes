3. Interne Tabelle

Definition auf Basis eines Strukturtyps

DATA lt_array TYPE TABLE OF strukturtyp.

Definition auf Basis eines Tabellentyps

DATA lt_array2 TYPE tabellentyp.

Initialisieren der internen Tabelle

REFRESH lt_array.

Anfügen eines Objektes an das Array:

APPEND ls_struktur to lt_array.

Einfügen mehrerer Objekte in eine interne Tabelle

*mit VALUE:
DATA(lt_array) = VALUE typ(
 ( komponente1 = 'WERT1' komponente2 = 'WERT2' )
 ( komponente1 = 'WERT1' komponente2 = 'WERT2' )
).

*über Datenbanktabelle
SELECT feld1 feld2
  FROM Datenbanktabelle
  INTO TABLE lt_array.
*(Array-Fetch-Technik)

Sortieren

SORT lt_array BY feld1 ASCENDING|DESCENDING.

Iteration über alle Objekte im Array durch Zuweisung einer Struktur

LOOP AT lt_array INTO DATA(ls_struktur).
 "Anweisungen"
ENDLOOP.

Iteration über alle Objekte im Array durch Zuweisung eines Zeigers (FIELD-SYMBOL)

LOOP AT lt_array ASSIGNING FIELD-SYMBOL(<fs_zeiger>).
 "Anweisungen"
ENDLOOP.

Lesen einer Zeile über Index

READ TABLE lt_array INTO ls_struktur
    INDEX index.
*oder neue Schreibweise:
ls_struktur = ls_array[ index ].

Lesen über bestimmte Kriterien

READ TABLE lt_array INTO ls_struktur
     WITH KEY feld1 = Vegleichsfeld.
*oder neue Schreibweise:
ls_struktur = ls_array[ feld1 = Vergleichsfeld ].

Modifikation eines Objektes innerhalb eines Arrays an bestimmter Stelle (Index):

MODIFY lt_array FROM ls_struktur INDEX index.

Löschen

DELETE lt_array INDEX index.
*Oder:
DELETE lt_array WHERE feld1 = Vergleichsfeld.

*Anzahl der Einträge in einer internen Tabelle
DESCRIBE TABLE lt_array LINES feld.
*oder
feld = lines( lt_array ).

Prüfung, ob ein Objekt mit bestimmtem Wert existiert

IF line_exists( lt_array[ feld1 = 'WERT' ] ).
 "Objekt gefunden
ENDIF.

Objektindex herausfinden

DATA(ln_index) = line_index( lt_array[ feld1 = 'WERT' ] ).

6.4. Manipulation des SALV

Methode Beschreibung

GET_SORTS

Liefert das Objekt der Klasse CL_SALV_SORTS zurück, mit dem Sie alle Sortierungsinformationen einstellen

GET_SELECTIONS

Liefert das Objekt der Klasse CL_SALV_SELECTIONS zurück, mit dem Sie die Markierungen der Tabelle angeben

GET_EVENT

Liefert das Objekt der Klasse CL_SALV_EVENT zurück, mit dem Sie die Ereignisse verwalten

GET_AGGREGATIONS

Liefert das Objekt der Klasse CL_SALV_AGGREGATIONS zurück, mit dem Sie Aggregationseinstellungen bearbeiten

GET_FILTERS

Liefert das Objekt der Klasse CL_SALV_FILTERS zurück, mit dem Sie alle Filtereinstellungen vornehmen

GET_DISPLAY_SETTINGS

Liefert das Objekt der Klasse CL_SALV_DISPLAY_SETTINGS zurück, mit dem Sie die Darstellung der Ausgabetabelle beeinflussen

GET_FUNCTIONAL_SETTINGS

Liefert das Objekt der Klasse CL_SALV_FUNCTIONAL_SETTINGS zurück, mit dem Sie vorwiegend das technische Umfeld Ihrer Ausgabetabelle verwalten

GET_LAYOUT

Liefert das Objekt der Klasse CL_SALV_LAYOUT zurück, mit dem Sie Einstellungen zur Layout-Verwaltung vornehmen

8.2. Befüllen der Knotentabelle

* Definition gt_nodes global:
* DATA: gt_nodes TYPE TABLE OF zsimpletree.
* (a) Prüfen ob Knotentabelle leer, nur wenn ja Weiterverarbeitung
  CHECK gt_nodes IS INITIAL.
* (b) Erzeugen Wurzelknoten
  CLEAR gs_node.
  gs_node-node_key = 'ROOT'.
  gs_node-isfolder = 'X'.
  gs_node-expander = 'X'.
  gs_node-text = 'X’.
  APPEND gs_node TO gt_nodes.

* (c) Lesen DB-Tabelle und einfügen in Knotentabelle
  CLEAR gs_node.
  gs_node-relatkey = 'ROOT'.
  gs_node-relatship = go_tree->relat_last_child.

* (d) Select-Statement durchführen, Füllen von gt_nodes

* (e) Aufruf Methode add_nodes
  go_tree->add_nodes(
  EXPORTING
    table_structure_name  = 'ZSIMPLETREE'    " Name der Struktur der Knotentabelle
    node_table            = gt_nodes         " Knotentabelle
).
