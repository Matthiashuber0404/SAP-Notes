1. Einführung

1.1. Begriff

Es gibt viele verschiedene Möglichkeiten, Werte ins System einzugeben. Dazu gehören einfache Eingabefelder, nummerische Eingaben, RadioButtons, Checkboxen, Dropdown-Felder, TextArea-Bereiche, usw.

Jedes Eingabeelement stellt ein eigenständiges Objekt mit eigenen Eigenschaften und Methoden dar.

1.2. Grundform

Damit mit Inputs gearbeitet werden kann, muss im XML-View folgende Eintragung gemacht werden:

xmlns:l="sap.ui.layout"

Dies ist eine SAPUI5- Bibliothek, welche zur Layout-Darstellung benötigt wird.

Zur Erstellung einer Select-Box ist folgende Ressource nötig.

xmlns:core="sap.ui.core"

Inputs können unterschiedlich dargestellt werden. In folgendem Beispiel wird das VerticalLayout verwendet.

<l:VerticalLayout class="sapUiContentPadding" width="100%">… 	</l:VerticalLayout>

Alle Eingaben werden zwischen diesen Tags durchgeführt.

1.3. EXKURS: MessageBox

Für die Ausgabe der Daten verwenden wir hier die MessageBox. Diese funktioniert ähnlich wie der MessageToast.

Sie muss als Ressource eingebunden und übergeben werden. Erst dann ist eine Verwendung möglich.

sap.ui.define([
	"sap/ui/core/mvc/Controller",
	"sap/m/MessageBox"
], function (Controller, MessageBox) {
	"use strict";
}

Es gibt veschiedene Methoden, die MessageBox aufzurufen:

MessageBox.alert("Ausgabe");
MessageBox.confirm("Bestätigung");
MessageBox.error("Fehler");
MessageBox.information("Information");
MessageBox.show("Anzeige");
MessageBox.success("Erfolg");
MessageBox.warning("Warnung");

Genauere Informationen zu allen Objekten können über die API Reference im Demo Kit (https://sapui5.hana.ondemand.com/) nachgeschaut werden.

2. Einfache Input-Felder

2.1. Text und Number

Jedes Input-Feld sollte ein Label besitzen. Das Label bezieht sich auf die ID des Input-Feldes.

text

<Label class="label" text="Name" labelFor="inpName"/>
<Input id="inpName" class="sapUiSmallMarginBottom" type="Text" placeholder="Enter Name ..."/>

number

<Label class="label" text="aktuelles Alter" labelFor="inpAlter"/>
<Input id="inpAlter" class="sapUiSmallMarginBottom" type="Number" placeholder="Alter"/>

2.2. Auslesen und verändern

Zunächst werden zwei Buttons im View erzeugt:

<footer>
    <Toolbar>
	    <ToolbarSpacer/>
	    <Button type="Accept" text="Ausgabe" press="onAccept"/>
		<Button type="Reject" text="Eingaben löschen" press="onClear"/>
	</Toolbar>
</footer>
Figure 2. Einfacher Input (Text, Number)

In der Methode onAccept wird das aktuelle View-Objekt geholt in einer Variable gespeichert. Anschließend werden die Inputfelder über Ihre ID geholt (Methode byId("fieldID").getValue()) und in Variablen abgelegt.

Dann wird das Ergebnis(result) zusammengesetzt und ausgegeben.

onAccept: function () {
    var oView = this.getView();
    var inpName  = oView.byId("inpName").getValue();
    var inpAlter = oView.byId("inpAlter").getValue();
    var result = inpName + " ist " + inpAlter;
    MessageBox.success(result);
}
Figure 3. Ergebnisausgabe

Zum Verändern wird die Methode setValue() verwendet:

onClear: function () {
	MessageBox.warning("Eingaben werden gelöscht");
	this.getView().byId("inpName").setValue("");
	this.getView().byId("inpAlter").setValue("");
}
Figure 4. Warnung und Löschen

2.3. Feldinhalte prüfen

Im View können dem Input-Tag unterschiedliche Eigenschaften mitgegeben werden.
valueStateText: Meldungstext, der ausgegeben wird, wenn bei der Eingabe ein Fehler gemacht wurde
value: Dem Wert wird ein Prüftyp mitgegeben, u.U. Bereiche, Mindest- oder Maximalangaben
change/liveChange: dem Attribut change/liveChange wird eine Methode zugewiesen, die bei Veränderung des Feldes aufgerufen wird; die Methode wird im zugehörigen Controller implementiert

Typen zur Plausibilitätsprüfung

Zur Prüfung gibt es derzeit 13 verschiedene Klassen. Eingabefelder können auf Boolean, Date, Float, Integer, String, usw. geprüft werden.
Alle diese Klassen basieren auf dem gleichen Namensbereich: sap.ui.model.type und können in der API Reference nachgelesen werden.

Beispiel

Die Input-Felder mit der id="inpName" und der id="inpAlter" sollen geprüft werden.

Schritt 1: Eintragung im View
<Input id="inpName" class="sapUiSmallMarginBottom"  placeholder="Enter Name ..."
	valueStateText="Der Name darf nicht leer, maximal 10 Stellen sein."
	value="{ path: '',
             type: 'sap.ui.model.type.String',
             constraints: { minLength: 1, maxLength: 10 } }"
	change="onChange"  required="true" />
// ...
<Input id="inpAlter" class="sapUiSmallMarginBottom"
	valueStateText="Es dürfen nur positive Zahlen bis 150 eingegeben werden"
	value="{ path: '',
             type: 'sap.ui.model.type.Integer',
             constraints: { minimum: 1, maximum: 150 } }"
	change="onChange" />

Mit valueStateText wird der Fehlertext festgelegt, der unter dem Eingabefeld erscheinen soll, wenn ein ungültiger Wert eingegeben wurde.
Bei value muss zunächst der path festgelegt werden (hier: leer). Dann ist der type, gegen den geprüft wird (z.B. sap.ui.model.type.String) anzugeben und gewünschten Einschränkungen (constraints). Die Parameter hängen vom jeweiligen Prüftyp ab.

Schritt 2: Eintragung im Controller

1) Definition einer privaten Methode onValidate()

_onValidate: function(event) {
	var oBinding = event.getBindingInfo("value");
	try {
		oBinding.type.validateValue(event.getValue());
		event.setValueState("Success");
		return true;
	} catch (ex) {
		event.setValueState("Error");
		return false;
	}
}

2) Definition der Eventhandlermethode onChange()

onChange: function(oInput) {
	var oEvent = oInput.getSource();
	return this._onValidate(oEvent);
}

Immer wenn der Wert in einem Eingabefeld verändert wurde, wird gepüft, ob der Wert gültig ist, entsprechend der Status gesetzt und wahr bzw. falsch zurückgegeben.

3) Methode onAccept() mit Prüfung vor Weiterverarbeitung

onAccept: function() {
	let oView = this.getView();
	if (this._onValidate(oView.byId("inpName")) === true &&
		this._onValidate(oView.byId("inpAlter")) === true) {
		let inpName = oView.byId("inpName").getValue();
		let inpAlter = oView.byId("inpAlter").getValue();
		MessageBox.success("Hallo " + inpName + "!" + "\nDu bist " + inpAlter + "Jahre alt.");
	} else {
		MessageBox.error("Fehler bei den Eingaben");
	}
},

3. Komplexe Typen

3.1. RadioButton

RadioButtons sind so geartet, dass nur ein Wert ausgewählt werden kann.
Zur Auswertung wurde eine Toolbar im Footer angelegt.

<Label class="label" text="Klasse" labelFor="klasseInput" />
<RadioButtonGroup id="klasseInput">
	<buttons>
		<RadioButton text="FS11" selected="true"></RadioButton>
		<RadioButton text="FS12" selected="false"></RadioButton>
	</buttons>
</RadioButtonGroup>
<!-- ... -->
<footer>
	<Toolbar>
		<Button type="Accept" text="Abschicken" press="onAccept"></Button>
	</Toolbar>
</footer>
Figure 7. Radio Buttons

Zugriff auf RadioButtons erfolgt über die Methode getSelectedButton(). In der Methode onAccept muss daher folgender Code erstellt werden.

var oKlasse = this.getView().byId("klasseInput");
var klasse = oKlasse.getSelectedButton().getText();
MessageBox.show("Formular wurde abgeschickt: " + klasse);

3.2. CheckBox

Bei Check-Boxen können mehrere Werte ausgewählt werden. Im folgenden Beispiel ist bereits ein Wert vorselektiert.

<Label class="label" text="Hobbys"/>
<CheckBox id="SchwimmenInput" text="Schwimmen" selected="false"/>
<CheckBox id="ZockenInput" text="Zocken" selected="true"/>
Figure 8. Checkbox

Die Methode OnAccept wird mit der Methode getSelected() erweitert. Da ja mehrere Werte ausgewählt werden, müssen diese in einem Array (checkArr) gespeichert werden.

var checkArr = [];
if (this.getView().byId("SchwimmenInput").getSelected()) {
	checkArr.push(this.getView().byId("SchwimmenInput").getText());
}
if (this.getView().byId("ZockenInput").getSelected()) {
	checkArr.push(this.getView().byId("ZockenInput").getText());
}
console.log(checkArr);

3.3. SelectBox

Die Select-Box funktioniert ähnlich wie die RadioButtons. Die Box hat eine eindeutige id.

<Select id="faecherInput">
	<core:Item key="java" text="Java"/>
	<core:Item key="web" text="Webprogrammierung"/>
	<core:Item key="abap" text="ABAP"/>
	<core:Item key="bwl" text="BWL"/>
</Select>
Figure 9. Select Box

Um den gewählten Eintrag auszulesen, ist die Methode _getSelectedItemText() zu verwenden.

var fach = this.getView().byId("faecherInput")._getSelectedItemText();
MessageBox.show("Dein Lieblingsfach : " + fach);

3.4. TextArea

Mehrzeilige Kommentare können über die TextArea erfasst werden. TextAreas eignen sich auch, um Ergebnisse auszugeben. In diesem Fall ist das property editable auf false zu setzen.

<Label class="label" text="Deine Eingaben:" labelFor="areaInput" />
<TextArea rows="5" id="areaInput" placeholder="Inhalt des Formulars" editable="false" />
Figure 10. Ergebnis mit TextArea

Die TextArea kann ganz einfach über die ID angesprochen werden. Mit der Methode setValue() erhät der Textbereich seinen Inhalt.

this.getView().byId("areaInput").setValue(
	"Klasse: " + klasse +
	"\nHobbys: " + checkArr +
	"\nLieblingsfach: " + fach
);

3.5. Zusätzliche Hinweise

  • Natürlich gibt es neben den genannten Input-Elementen noch weitere Typen und Darstellungsformen.

  • Der Zugriff auf die einzelnen Input-Objekte muss abhängig ihrer definierten Eigenschaften/Methoden erfolgen.

    • Um diese nachsehen zu können, kann entweder

      • Die Hilfe verwendet werden oder

      • Man lässt sich das aktuelle Objekt über console.log() anzeigen alle Attribute und Methoden (Prototyp) werden aufgelistet

  • Berechnungen

    • Zur Berechnung können die üblichen Operatoren (+ - * / %) verwendet werden.

    • Mathematische Gesetzmäßigkeiten („Punkt vor Strich“) haben Gültigkeit, auch Klammern werden entsprechend aufgelöst.

    • Für besondere Berechnungen, wie Potenz, Wurzel, Sinus- und Cosinus, usw., kann die Klasse Math verwendet werden.
