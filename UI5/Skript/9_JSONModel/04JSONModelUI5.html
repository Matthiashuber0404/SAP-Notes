1. Arbeit mit Mockdaten

1.1. Anlage eines JSON-Files

Mockdaten sind Testdaten, sie werden in eigenständigen JSON-Files abgespeichert, i.d.R. im Verzeichnis model.

Das Dokument school.json (in Verzeichnis model) enthält drei Objekte (SCHOOL,CLASS, PUPIL). Jedes dieser Objekte enthält einen Array an Objekten. Unten sind einige Objekte zu sehen.

	"SCHOOL": [{ ... },
        {
        "SID": "FS",
		"SNAME": "Fachschule",
		"LNAME": "Fachschule für Wirtschaftsinformatik"
    }],
	"CLASS": [{ ... }, ...
	    {
		"SID": "FS",
		"CID": "12",
		"CDESC": "Staatlich geprüfter Wirtschaftsinformatiker"
	}],
	"PUPIL": [{
		"PID": 1,
		"SURNAME": "Adam",
		"FIRSTNAME": "Otto",
		"CITY": "München",
		"STYPE": "BFS",
		"CID": "10a",
		"DATEOFBIRTH": "20040101",
		"ENTRYDATE": "20200907"
	}, ...
    {

Das komplette JSON-Objekt kann über Laufwerk O: in Ihr Projekt übernommen werden.

1.2. Model erzeugen

Sollen Daten in JSON-Form verwendet werden, muss zunächst ein Model erzeugt und mit den Daten aus dem JSON-File gefüllt werden. Dies Erweitern Sie das File models.js um folgende Methode:

createSchoolModel: function () {
	var oModel = new JSONModel();
	oModel.loadData("model/school.json");
	return oModel;
}

In der component.js muss dann das Model gesetzt werden. Dazu rufen die die erstellte Methode createSchoolModel auf. Dem Schulmodel wird der Name school zugewiesen:

this.setModel(models.createSchoolModel(), "school");

Nun kann es verwendet werden.

2. Zugriff im XML-View

Der Zugriff über einen XML-View erfolgt über das Data-Binding der Modeldaten. Dazu wird eine passende Aggregation verwendet.

In unserem Beispiel sollen in einer Tabelle zunächst alle Schüler angezeigt werden. Das entsprechende Attribut/die Aggregation heißt items.

<Page id="page" title="{i18n>title}">
	<content>
		<Table headerText="Schülerübersicht" id="tPupils" items="{ path: 'school>/PUPIL'  }"> ...
        </Table>
    </content>
</Page>

Bei dem items wird ein Objekt angegeben ( {} ), welches den Pfad der Daten beinhaltet: Modelname>/Objektname.

Zwischen dem Start- und Ende-Tag der Table werden
a) Spaltenüberschriften festgelegt (<columns>)
b) Komponenten auf die Spalten zugeordnet (<items>)

<Table headerText="Schülerübersicht" id="tPupils" items="{ path: 'school>/PUPIL'  }">
	<columns>
		<Column>
			<header>
				<Label text="Vorname"></Label>
			</header>
		</Column>
		<Column>
			<header>
				<Label text="Nachname"></Label>
			</header>
		</Column>
		<Column>
			<header>
				<Label text="Schultyp"></Label>
			</header>
		</Column>
		<Column>
			<header>
				<Label text="Klasse"></Label>
			</header>
		</Column>
	</columns>
	<items>
		<ColumnListItem>
			<cells>
				<Text text="{school>FIRSTNAME}"/>
				<Text text="{school>SURNAME}"/>
				<Text text="{school>STYPE}"/>
				<Text text="{school>CID}"/>
			</cells>
		</ColumnListItem>
	</items>
</Table>

Auf einzelne Eigenschaften wird folgendermaßen zugegriffen: Modelname>Property (Beispiel: {school>FIRSTNAME})

3. Filter und Sortierung

3.1. Suchen und Filtern

Ein Suchfeld soll eingefügt werden. Darin kann eine Klasse eingegeben werden. Abhängig vom eingegebenen Wert sollen nur Schüler der eingegebenen Jahrgangsstufe angezeigt werden.

Im View muss dazu nach dem Start-Tag Table folgende Erweiterung durchgeführt werden:

<headerToolbar>
	<Toolbar>
		<ToolbarSpacer />
		<Title text="Klasse"></Title>
		<SearchField id="searchClass" width="20%" search="onSearch"></SearchField>
	</Toolbar>
</headerToolbar>

Dies bewirkt, dass die Tabelle eine Suchleiste im Header erhält:

Figure 2. Search Field

Im Controller des Views kann nun über den Wert ein Filter gesetzt werden:
Dazu müssen folgende Ressourcen eingebunden werden:

sap.ui.define([
	"sap/ui/core/mvc/Controller",
	"sap/m/MessageBox",
	"sap/ui/model/Filter",
	"sap/ui/model/FilterOperator"
], function (Controller, MessageBox, Filter, FilterOperator) {
onSearch: function (oEvent) {
	var sQuery = oEvent.getParameter("query");	// 1)
	console.log(sQuery);
	var aFilter = [];							// 2)
	if (sQuery) {								// 3)
		aFilter.push(new Filter("CID", FilterOperator.Contains, sQuery));
	}
	var oTable = this.byId("tPupils");			// 4)
	var oBinding = oTable.getBinding("items");	// 5)
	oBinding.filter(aFilter);					// 6)
}

Erläuterung:
1) Holen des Eingabewertes: in sQuery steht die eingegebene Klasse.
2) Definition eines Filter-Arrays
3) Filter setzen: Angabe des Feldes (KID) und des Operators (EQ(equal), Contains, usw.)
Filter an Tabelle binden:
4) Tabellenobjekt über id holen
5) Binding zu den items erzeugen
6) Items mit Filter verknüpfen

Wird im Suchfeld beispielsweise 10 eingegeben und die Lupe gedrückt, sollte das Ergebnis folgendermaßen aussehen:

Figure 3. Search Field mit Wert

Bei Drücken von x wird das Suchfeld gelöscht und wieder alle Sätze angezeigt.

3.2. Sortierung

Einfachste Variante: Attribut sorter zu items hinzufügen:

<Table headerText="Schülerübersicht"
    id="tPupils"
    items="{ path: 'school>/PUPIL' , sorter: { path: 'SURNAME' } }">

Dann sind die Daten sortiert, ganz ohne JavaScript!

4. Aggregation im Tile

4.1. Füllen der Tiles

Tiles können selbstverständlich auch mittels externer Datenquelle befüllt werden.

Im folgenden Beispiel wird das JSON-Objekt SCHOOL verwendet:

<FlexBox width="100%" items="{ path: 'school>/SCHOOL' }">
	<items>
		<GenericTile header= "{school>SID}" frameType="Auto" press="onTile">
			<TileContent footer="{school>SNAME}">
				<Text text="{school>LNAME}" />
			</TileContent>
		</GenericTile>
	</items>
</FlexBox>
Figure 5. Tiles über Model befüllt

4.2. Klickevent verarbeiten

Wird ein Tile gewählt, dann wird die Methode onTile aufgerufen. Das Event liefert das aktuelle Objekt in den Controller mit und kann über geeignete Methoden ausgelesen werden.

Hier eignet sich die Methode getHeader(), welche den (alphanumerischen) Primärschlüssel liefert.

onTile: function (oEvent) {
	var sQuery = oEvent.getSource().getHeader();
	console.log(sQuery);
	var aFilter = [];
	if (sQuery) {
		aFilter.push(new Filter("STYPE", FilterOperator.EQ, sQuery));
	}
	var oTable = this.byId("tPupils");
    var oBinding = oTable.getBinding("items");
	oBinding.filter(aFilter);
}

Anhand dieses Primärschlüssels kann nun ebenso wie oben ein Filter auf die Tabelle gesetzt werden.

Einziger Unterschied: Sie filtern auf die Komponente STYPE im PUPIL-Objekt und verwenden den Filter-Operator EQ (EQUAL).

Figure 6. Filter über Schultyp

Vorsicht: Bisher ist nur ein Filter möglich, d.h. man kann entweder über Schultyp oder Klasse filtern. Eine Kombination geht bisher noch nicht.

5. Zugriff auf Model im Controller

Sehr oft muss auch im Controller auf die Daten des Models zugegriffen werden.

Dazu wird zunächst im View ein Button Information im footer eingefügt:

<footer>
	<Toolbar>
	    <Button id="idInfo" text="Information" type="Default" press="onInfo"></Button>
	</Toolbar>
</footer>

Bei Klick auf den Button sollen in einer MessageBox die Anzahl der Schulen, Klassen und Schüler angezeigt werden.

onInfo: function () {
	var aPupilJson = this.getView().getModel("school");				// 1)
	var obj = JSON.parse(aPupilJson.getJSON());						// 2)
	MessageBox.information("Anzahl Schulen: " + obj.SCHOOL.length +	// 3)
				"\nAnzahl Klassen: " + obj.CLASS.length +
				"\nAnzahl Schüler: " + obj.PUPIL.length);
		}
Erläuterung

1) Das Datenmodell „school“ wird geholt und in der Objektvariable aPupilJson gespeichert. Dies kann man mit Hilfe der Console nachvollziehen.

2) Die Daten werden zur Verarbeitung mit der Methode getJSON() geholt und in JSON umgewandelt (JSON.parse).
