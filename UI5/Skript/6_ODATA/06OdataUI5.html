2.2. Test READ: SAP Gateway Client

Über die Transaktion /IWFND/MAINT_SERVICE gelangt man in die Serviceverwaltung.

Hier kann der SAP Gateway Client aufgerufen werden, ein Tool zum Testen des Webservices.

Grundsätzlich sollte immer erst jede Methode Ihres Webservices auf diese Weise getestet werden. Erst dann ist die Verwendung im Frontend sinnvoll.

Treten hier schon Fehler auf, dann liegt das Problem in der Implementierung der Methode. Zur Erleichterung der Fehlersuche können Sie sich einen externen Haltepunkt setzen.

Durch Auswahl des EntitySets und der HTTP-Methode GET (+ F8) wird die Methode pupilset_get_entityset aufgerufen und das Ergebnis angezeigt.

3. einen Satz lesen (READ)

3.1. Backend

Um einen Satz vom Backend lesen zu können, muss die Methode pupilset_get_entity redefiniert werden.

Tip
 Man braucht diese Methode auch, wenn Sätze mit Update geändert oder mit Delete gelöscht werden sollen. Beide Methode rufen vorher die Methode zum Lesen eines einzelnen Satzes auf! 
  METHOD pupilset_get_entity.
    DATA: ls_key_tab LIKE LINE OF it_key_tab,
          ls_pupil   TYPE z001_63_s_pupil,
          lv_message TYPE bapi_msg.

******************************
* Lesen des Satzes
******************************
    IF it_key_tab IS NOT INITIAL.
      "READ TABLE it_key_tab INTO ls_key_tab WITH KEY name = 'Id'.
	  ls_key_tab = it_key_tab[ name = 'Id' ].

      IF ls_key_tab IS NOT INITIAL.
        SELECT SINGLE id surname name cid groupno FROM z001_63_pupil INTO ls_pupil
          WHERE id = ls_key_tab-value.

******************************
* Ergebnisbehandlung
******************************
        IF sy-subrc <> 0.
          RAISE EXCEPTION TYPE /iwbep/cx_mgw_busi_exception
            EXPORTING
              message = lv_message.
        ELSE.
          MOVE-CORRESPONDING ls_pupil TO er_entity.
        ENDIF.
      ENDIF.
    ENDIF.
  ENDMETHOD.

Welcher Satz gelesen werden soll, wird in der internen Tabelle it_key_tab (Importparameter) übermittelt. In dieser Tabelle stehen Wertepaare, die Attribute und Attributwerte beinhalten.

Beispielsweise soll über die Schüler-Id gelesen werden, dann wird als Attribut (name) der Komponentenname Id sowie die konkrete Ausprägung (value) gesendet.

onInit: function () {
	var oRouter = new sap.ui.core.UIComponent.getRouterFor(this);
	oRouter.getRoute("Detail").attachPatternMatched(this._onObjectMatched, this);
},
_onObjectMatched: function (oEvent) {
	var sQuery = oEvent.getParameter("arguments").detailPath;

	var that = this;
	var oModel = this.getView().getModel("schueler");
	var pupilId = parseInt(sQuery);
	var sPath = "/pupilSet(" + pupilId + ")";

	var fnSuccess = function(data){
		that.getView().byId("idInput").setValue(data.Id);
		that.getView().byId("nachnameInput").setValue(data.Surname);
		that.getView().byId("vornameInput").setValue(data.Name);
		that.getView().byId("klasseInput").setValue(data.Cid);
		that.getView().byId("gruppeInput").setValue(data.Groupno);
	}
	var fnError = function () {
		MessageBox.error("FEHLER: Schüler gibt es nicht");
	}
	var mSettings = {
		success: fnSuccess,
		error: fnError
	};
	oModel.read(sPath, mSettings);
}

7. READ (mit Filter)

Soll das Ergebnis serverseitig nach beliebigen Komponentenwerten gefiltert werden, muss die Backendmethode get_entityset angesteuert und dort der Filter durchgeführt werden.

Frontentseitig wird der Methode read der gewünschte Filter mitgegeben. Das kann ein Filter sein oder auch mehrere.

7.1. Frontend

Im View werden die Filter eingelesen:

<Label text="Klassen-Id"></Label>
<Input id="idKlasseInput" width="100px"></Input>
<Label text="Gruppe"></Label>
<Input id="idGruppeInput" width="100px"></Input>
<Button text="Lesen mit Filter" press="onReadWithFilter"/>

7.1.1. Alternative 1

Die Daten werden nicht im View geholt, sondern über die Methode read. Daher muss man der Methode read den Filter mitgeben.

Vorgehen

1) In den Controller werden die folgenen Bibliotheken eingebunden:

   "sap/ui/model/Filter",
   "sap/ui/model/FilterOperator"

2) Definition eines Filterarrays:

	var aFilters = [];

3) Aus den gewünschten Filterwerten werden Filter erzeugt:
Beispiel: Der Wert des Inputfeldes mit der id="nachnameInput" wird zum Filter:

   var filter1 = new Filter({
		path: "Surname", operator: FilterOperator.EQ, value1: nachname
	});
	Erklärung
	path    : Name des Service-Attributs
	operator: Es gibt verschiedene Optionen,
	          z.B. EQ (equal), LE (less or equal), BT (between), usw.
	value1  : aktueller Wert, z.B. aus Frontend ( hier: nachname )
	value2  : optional, wird benötigt, wenn als Operator = BT verwendet wurde

4) Der Filter wird in das Filterarray hinzugefügt (push).

5) Die Filter werden dann der read-Methode mitgegeben.Im folgenden Beispiel werden zwei Filter verwendet.

onReadWithFilter: function(){
	var oModel = this.getView().getModel("schueler");
	let sKlasseId = this.byId("idKlasseInput").getValue()
	let sGruppeId = this.byId("idGruppeInput").getValue()
	var sPath = "/pupilSet";
	let aFilter = []
	if(sKlasseId){
		aFilter.push(new Filter("Cid", FilterOperator.EQ, sKlasseId))
	}
	if(sGruppeId){
		aFilter.push(new Filter("Groupno", FilterOperator.EQ, sGruppeId))
	}

	var fnSuccess = function (data) {
		MessageBox.show("Schüler gefunden");
		console.log(data);
	}
	var fnError = function () {
		MessageBox.error("FEHLER: Keine Datensätze zum Filter gefunden");
	}
	oModel.read(sPath, {
		filters: aFilter,
		success: fnSuccess,
		error: fnError
	});

7.2. Backend

Backendseitig muss der Filter ausgelesen werden. Das geschieht über die Methode get_entityset und den Parameter it_filter_select_options.

METHOD pupilset_get_entityset.

   	DATA: lt_filter_cid TYPE /iwbep/t_cod_select_options.
	DATA: lt_filter_groupno TYPE /iwbep/t_cod_select_options.
	DATA: lv_where TYPE string.

	lv_where = ||.
	LOOP AT it_filter_select_options ASSIGNING FIELD-SYMBOL(<fs_filter>).
		CASE <fs_filter>-property.
		WHEN 'Cid'.
			lt_filter_cid = <fs_filter>-select_options.
		WHEN 'Groupno'.
			lt_filter_groupno = <fs_filter>-select_options.
		ENDCASE.
	ENDLOOP.

	IF lines( lt_filter_cid ) > 0.
		lv_where = |{ lv_where } and cid IN lt_filter_cid|.
	ENDIF.

	IF lines( lt_filter_groupno ) > 0.
		lv_where = |{ lv_where } and groupno IN lt_filter_groupno|.
	ENDIF.

	REPLACE FIRST OCCURRENCE OF 'and' IN lv_where WITH space.

	SELECT * FROM z001_63_pupil
			INTO TABLE et_entityset
			WHERE (lv_where).

  ENDMETHOD.
Erklärung

Die Filter aus dem Frontend kommen in der internen Tabelle it_filter_select_options an. In der LOOP-Schleife werden die Filter einzeln ausgelesen und in einem weiteren Schritt in lokale Variablen gespeichert. Dann können über die SELECT-Anweisung die gewünschten Datensätze gelesen werden.

Note
 Es kann mit alle beliebigen Filtern gearbeitet werden. Das Ergebnis können mehrere Datensätze sein, oder nur Einer oder das Ergebnis ist leer. Im Unterschied dazu muss bei get_entity immer mit dem Primärschlüssel zugegriffen werden. Das ist hier nicht nötig!
